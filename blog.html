Use of Relative Size When Styling a Responsive Element

March 12 2020


Responsive web design allows for elements to be automatically changed so that everything fits properly in all types of devices. Responsive elements are able to automatically resize (shrink or enlarge), hide and/or move. This short blog will focus on the pros and cons of using relative size units in css (%, em, rem) to create an element responsive to its device.


While it is not exactly a requirement or necessary to use responsive elements in terms of size as most browsers have incorporated zoom features, there are many benefits to responsive components approach (both at the designer/developer and user-end).


A few reasons why responsive elements are beneficial . . .

  • It is more logical for designers/developers to focus on the sizes of elements relative to screen size and in proportion to each other rather than in specific size units.
  • Responsive elements will ensure a site looks well put together on a screen of any size. They may also ensure that a site is mobile-friendly.
  • Components in the site will always be in proportion.

    March 13 2020


    One may wonder about how browsers interpret custom font-sizes. In almost every browser, the standard root element font-size is assumed to be 16px.

    As a result, when one sets a custom font-size, be it 200px or any other size really, the browser interprets that size in terms of the root element size 16px.

    Changes in CSS3

    March 17 2020


    This blog update will discuss some of the many changes that have been made in CSS3 briefly by picking the most important ones.


    on Backgrounds, Borders and more: many available options to play with . . .

  • multiple backgrounds - multiple images can be set on the background
  • image border - images can be set as borders
  • rounded corners - new "border-radius" property allows for rounded corners
  • opacity - elements can be made see-through or even completely transparent
  • miscellaneous - new colors and fonts added


    • on Relative Sizing: CSS3 provided designers and developers with the new alternative relative font-sizing units based on viewport dimensions . . .

  • vw - viewport width (1vw = 1% of viewport width)
  • vh - viewport height (1vh = 1% of viewport height)


    • on Grid Layout: CSS3 allows web designers/developers to build a page layout using a two dimensional system that can deal with both rows and columns . . .

  • explicit grids - area defined with "grid-template-rows" and "grid-template-columns"
  • implicit grids - automatically generated when defined grids are less than actual grids
  • flexible lengths (NEW) - new "fr" unit introduced that allows for free space remaining in the grid container to be viewed when checked


    • on Flexible Box: As opposed to a box, the flexible box adapts the layout of the page according to the diverse screen sizes and display gadgets . . .

  • does not use floats (cleaner and simpler in terms of usage)
  • container margin does not fall apart (doesn't depend on margin of information)


    • on Media Queries: designed to inherently support responsive design
